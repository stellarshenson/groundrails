# SVG Infographics - Correction Forensics

Forensics of one production session: five infographics generated by `svg-designer` agents (svg-infographics plugin), followed by 14 human-requested corrections. Every correction was caught by eye on a Playwright render - none by `check`/`finalize`, which passed (or ack-passed) all five files first. Handover notes for plugin plumbing.

## What the validators missed vs flagged

- **Missed entirely** - 1, 2, 3, 6, 7, 8, 11, 12, 13, 14 (geometry/convention defects, all visible on first render)
- **Flagged but acked away** - 4 (real overlap classified "accepted design")
- **False-positive noise that trained the ack reflex** - icon paths inside `scale()` groups scored as routed connectors (parent transform not applied); background rect classified as a card; `@media (prefers-color-scheme: dark)` fills not parsed → "missing dark override" warnings on every file; adjacent grid cells flagged as overlaps unless `--raw`

## Plumbing suggestions

- **Render-based gate** - screenshot via Playwright + geometry on the *rendered* DOM (real text widths) as a finalize stage; every one of the 14 defects was visible on first render
- **Real text metrics in checkers** - the Opus fix agent measured title widths with fontTools; promote that into `overlaps` so text-vs-icon collisions stop passing
- **Corner-placement primitive** - `place --corner top-right --pad 8 --from accent-bar` for icons/badges/digits; three correction rounds existed only because this rule lived in nobody's head
- **Canvas-balance check** - flag >10% dead canvas on one side and connector stems below a minimum length
- **Manifold detection** - same source, N targets → suggest manifold; offer spline as a style for bipartite maps
- **Card anatomy card** - one rule card pinning square-top/rounded-bottom bodies, accent bar, icon slot, stat slot parity - and have `check` verify it per card grid
- **Fix the false-positive trio** - apply parent transforms before connector classification, parse `@media` blocks, exempt the background rect; the noise is what makes agents ack real findings (correction 4)
- **Stale-element refresh in `callouts`** - occupancy snapshot should rebuild after edits, or NO FIT results mislead
